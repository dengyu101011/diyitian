创建一个分支：git branch 分支名。如 git branch testing
git branch 表示列出当前都有哪些分支
默认情况下，并不会切换到新创建的分支上面；git checkout 分支名 此时就切换到testing分支了
删除分支 
git branch -d 分支名   如：git branch -d testing  他能删除的是刚创建的，里面没有内容的 或者 合并已经合并到master分支了
如果一个分支中有内容并且没有合并到master分支，要删除 通过：git branch -D 分支名
如果要删除一个分支，必须要切换到master分支
分支切换会改变工作区的文件，记住：在切换分支之前先提交一下形成版本。
合并分支：
    通过git merge 分支名
    只有在master分枝上才能合并其他分支
    git clone  远程仓库的地址

    工作流：
        开发项目。
        每个程序员，都会创建一个属于自己的分支。
        在自己的分支上开展工作。
        》》测试组对你，主分支（一般都是将要发布或线上版本）上面有一个bug，需要解决，bug是56# bug
        在创建一个分支去修复这个bug,然后，你就创建一个分支（debug56），去修bug....
        》》老板说，主分支上有一个非常严重的bug,你赶紧修改一下。
        在创建一个分支去修复这个严重的bug,分支叫（hotbug），去修bug....
        当严重的bug修复ok了，把hotbug分支合并到主分支上
